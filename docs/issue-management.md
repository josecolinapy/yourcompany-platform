# Gestión de issues

## Regla principal
Todo cambio relevante debe estar asociado a una issue.

## Tipos recomendados
- `feature`
- `bug`
- `refactor`
- `technical-debt`
- `documentation`
- `security`
- `performance`

## Qué debe tener una feature issue
- Resumen
- Problema
- Alcance
- Fuera de alcance
- Criterios de aceptación
- Notas técnicas
- Dependencias

## Qué debe tener una bug issue
- Resumen
- Comportamiento actual
- Comportamiento esperado
- Pasos para reproducir
- Evidencia
- Entorno
- Impacto

## Definition of Ready
Una issue está lista cuando:
- Tiene objetivo claro
- Alcance claro
- Criterios de aceptación
- Dependencias conocidas

## Definition of Done
Una issue está completa cuando:
- Código implementado
- Tests actualizados
- Build verde
- PR mergeada
- Documentación actualizada si aplica
