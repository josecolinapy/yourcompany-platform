# Política de seguridad

## Cómo reportar vulnerabilidades

Si encuentras una vulnerabilidad de seguridad:

1. No abras una issue pública con el detalle sensible;
2. Utiliza el canal interno o privado definido por la organización;
3. Incluye evidencia mínima para reproducir;
4. Indica impacto estimado y componente afectado.

## Qué se considera sensible

- Credenciales
- Tokens
- Llaves privadas
- Secretos de integración
- Datos personales sensibles
- Configuraciones inseguras explotables

## Buenas prácticas para contribuidores

- Nunca commitear secretos
- No loggear datos sensibles
- Revisar impacto de cambios en crypto, security
