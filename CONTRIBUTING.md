# Contribuir

Gracias por contribuir a esta plataforma.

## Reglas básicas

- Código en inglés
- Documentación y templates en español
- Cambios relevantes deben tener issue
- Commits con Conventional Commits
- PR obligatoria para cambios relevantes
- Tests y documentación actualizados si el cambio lo requiere

## Validación mínima

```bash
mvn clean verify
```

## Antes de abrir PR
- revisar `docs/coding-standard.md`
- revisar `docs/git-workflow.md`
- revisar `docs/versioning.md`

## Qué se espera de una contribución
- Claridad
- Trazabilidad
- Reutilización
- Bajo acoplamiento
- Consistencia con la plataforma
